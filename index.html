Что такое стайл-гайд для фронтенд

  1. Результат объединения работы UX-дизайнеров и разработчиков
  2. Модульнуя коллекцию всех элементов пользовательского интерфейса
  3. Фрагменты кода с примерами

Ключевые элементы стайл-гайд

  1. Содержание, которое предоставляет легкую навигацию по компонентам
  2. Цветовая палитра (в соответствующем формате, HEX для Web, UIColor для iOS app, и т.д.)
  3. Типографские стили
  4. Отзывчивая разметка (сетка), чтобы красиво разместить элементы
    (это может, скорее даже должна быть та же сетка, что вы используете в проекте)

Описание компонентов

  1. Описание подходящего контекста использования: когда нужно использовать данный компонент, а не любой иной сходный с ним?
  2. Примеры кода
  3. Спецификацию для имплементации, включая информацию о позиционировании
  4. За и против для использования этого элемента (опционально)

Базовый лист компонентов

  • Buttons
  • Button groups
  • Breadcrumbs
  • Cards
  • Tables
  • Dialogs
  • Grid lists of content, media, or photos
  • Vertical lists
  • Navigation menus (and subnavigation)
  • Date and time pickers
  • Progress and loading indicators
  • Checkboxes
  • Radio buttons
  • Drop-down menus
  • Sliders
  • On-off switches
  • Numeric-input steppers/incrementers
  • Form fields (include variations with maximum character-count indicators, and indicators for when the field is required)
  • Tabs
  • Toolbars
  • Tooltips
  • Alert modals
  • Icons
  • Animations
  • Tokens, also known as chips (e.g. email-address field)
